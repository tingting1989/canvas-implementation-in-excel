### 2️⃣ TileRenderer 集成点完善 ⭐⭐⭐

#### 修改位置
[custom-renderer-api.md#L518-L619](./custom-renderer-api.md#L518-L619)

#### 修改前
```javascript
#createRenderContext(sheet, r, c, cell, drawX, drawY, w, h, merge) {
    return new CellRenderContext({
        row: r,
        col: c,
        // ❌ 缺少 realRow/realCol
        // ❌ 缺少 sheet 引用
        // ❌ 缺少 pageInfo
    });
}
```

#### 修改后
```javascript
#createRenderContext(sheet, r, c, cell, drawX, drawY, w, h, merge, options) {
    // ★★★ 核心：计算实际行号（与图表引擎对齐）★★★
    const realR = (options?.useRealRows) ? r : sheet.toRealRow(r);
    const realC = c;

    // 构建分页/冻结信息
    const pageInfo = {
        isPaged: sheet.isPagedMode(),
        currentPage: sheet.getCurrentPage?.() ?? 0,
        pageSize: rc.getPageSize?.() ?? 0,
        frozenRowCount: rc.frozenRowCount || 0,
        frozenColCount: rc.frozenColCount || 0,
        isInFrozenArea: r < (rc.frozenRowCount || 0),
    };

    return new CellRenderContext({
        sheet: sheet,           // ✅ 新增
        row: r, col: c,
        realRow: realR,         // ✅ 新增
        realCol: realC,         // ✅ 新增
        pageInfo: pageInfo,     // ✅ 新增
    });
}
```

#### 关键改进
- ✅ 实现双轨行列号计算逻辑
- ✅ 集成现有的 `sheet.toRealRow()` 方法
- ✅ 构建 PageInfo 对象（从 RowColManager 获取）
- ✅ 支持分页模式的 `useRealRows` 选项

## 🎯 解决的核心问题

### 问题1：冻结场景下渲染错位 ✅ 已解决

**原因**：原文档未区分实际行号和页面行号
**解决方案**：
- 提供 `realRow` 和 `row` 双属性
- 详细说明冻结区域的行号映射关系
- 给出正确的代码示例和常见陷阱

### 问题2：分页模式下数据混乱 ✅ 已解决

**原因**：原文档未考虑分页模式的行号偏移
**解决方案**：
- 提供 `pageInfo` 对象封装分页状态
- 处理分页+冻结的最复杂组合场景
- 给出绝对实际行号的计算方法

### 问题3：高级场景受限 ✅ 已解决

**原因**：Context 缺少 Sheet 引用
**解决方案**：
- 开放 `sheet` 属性访问完整 Sheet API
- 示例展示跨单元格数据读取
- 支持事件监听和数据联动

### 问题4：与图表引擎不一致 ✅ 已解决

**原因**：两套系统使用不同的行列号约定
**解决方案**：
- 统一采用"存储用实际行号，显示用页面行号"原则
